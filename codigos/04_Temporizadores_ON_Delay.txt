EXPLICACIÓN DETALLADA DEL PROGRAMA LADDER CON TEMPORIZADOR TON
PLC HORNER HE-XC1E5 XL4 – CSCAPE 10.2

1. CONTEXTO GENERAL DEL PROGRAMA

El programa mostrado corresponde a un diagrama ladder, también conocido como código en escalera. Este lenguaje se utiliza para programar PLC, es decir, programadores lógicos programables, que son equipos usados en automatización industrial para controlar entradas, salidas, marcas internas, temporizadores, contadores y diferentes condiciones lógicas.

En este caso, el programa está desarrollado en Cscape 10.2, que es el software utilizado para programar PLC de la marca Horner, específicamente el modelo de la serie HE-XC1E5 XL4.

El ladder se interpreta como si fuera un circuito eléctrico de control. La corriente lógica fluye de izquierda a derecha. Si las condiciones colocadas en una línea se cumplen, la corriente puede avanzar hasta activar una bobina, una marca interna, una salida física o un bloque de función.

Este programa es muy parecido al programa anterior del temporizador TOF, pero con una diferencia importante: en este caso se utiliza un temporizador TON. El TON funciona como un temporizador con retardo a la conexión, es decir, primero espera un tiempo determinado y después activa su salida.

La finalidad de este programa es encender la lámpara amarilla Q11 después de que el botón de arranque I1 se mantenga presionado durante 5 segundos. Si el botón se suelta antes de que pasen los 5 segundos, el temporizador se reinicia y la lámpara no se enciende.

2. ASIGNACIÓN GENERAL DE ENTRADAS Y SALIDAS DEL PLC

Para este PLC Horner HE-XC1E5 XL4 se consideran las siguientes entradas físicas del tablero:

- I1: Botón normalmente abierto, utilizado como botón de arranque.
- I2: Botón normalmente cerrado, utilizado comúnmente como botón de paro.
- I3: Entrada normalmente abierta asociada a un selector o botón auxiliar.
- I4: Entrada normalmente abierta asociada a otra posición del selector.
- I8: Botón de paro de emergencia normalmente cerrado.

También se consideran las siguientes salidas físicas para las lámparas del tablero:

- Q10: Lámpara verde.
- Q11: Lámpara amarilla.
- Q12: Lámpara roja.

En este programa se utiliza específicamente la salida Q11, por lo tanto, la lámpara controlada es la lámpara amarilla.

3. ELEMENTOS PRINCIPALES DEL PROGRAMA

En la imagen del programa ladder se pueden identificar los siguientes elementos:

- Contacto normalmente abierto %I0001.
- Temporizador TON asociado al registro %R0001.
- Base de tiempo configurada en 0.1 segundos.
- Parámetro PT configurado en 50.
- Bobina interna %M00001.
- Contacto normalmente abierto %M00001.
- Bobina de salida física %Q0011.

Cada uno de estos elementos cumple una función dentro de la lógica del programa.

4. ¿QUÉ ES UN TEMPORIZADOR EN LADDER?

Un temporizador es un bloque de función que permite controlar acciones dependiendo del tiempo.

En automatización, los temporizadores son muy importantes porque permiten que una salida no dependa únicamente de que una entrada se active, sino también de que transcurra un tiempo específico.

Los temporizadores pueden servir para:

- Encender una lámpara después de cierto tiempo.
- Apagar una salida después de un retardo.
- Mantener un motor funcionando por un tiempo definido.
- Crear pausas entre etapas de una secuencia.
- Activar una alarma después de que una condición permanezca activa.
- Evitar activaciones instantáneas por señales momentáneas.
- Controlar procesos que necesitan esperar antes de ejecutar una acción.

En este programa, el temporizador se utiliza para que la lámpara amarilla no se encienda inmediatamente al presionar el botón, sino únicamente después de que el botón I1 se mantenga presionado durante 5 segundos.

5. BLOQUE TON – TIMER ON DELAY

El bloque utilizado en este programa es un temporizador tipo TON.

TON significa Timer On Delay, que en español se puede interpretar como temporizador con retardo a la conexión o temporizador de encendido retardado.

Este tipo de temporizador funciona de la siguiente manera:

- Cuando la entrada del temporizador se activa, el temporizador empieza a contar.
- La salida del temporizador no se activa inmediatamente.
- La entrada debe permanecer activa durante todo el tiempo configurado.
- Cuando el tiempo acumulado llega al valor preestablecido, la salida del temporizador se activa.
- Si la entrada se desactiva antes de llegar al tiempo configurado, el temporizador se reinicia.
- Cuando la entrada se desactiva después de que el temporizador ya activó su salida, la salida se apaga y el tiempo vuelve a cero.

Por esta razón se llama temporizador de retardo a la conexión, porque retrasa el momento en el que se activa la salida.

6. DIFERENCIA ENTRE TON Y TOF

Este programa es similar al programa anterior, pero el tipo de temporizador cambia el comportamiento del sistema.

En el programa anterior se utilizaba un TOF, que significa Timer Off Delay. El TOF activa la salida de forma inmediata cuando recibe la señal y después retrasa el apagado cuando la señal desaparece.

En este programa se utiliza un TON, que significa Timer On Delay. El TON no activa la salida inmediatamente. Primero espera el tiempo configurado y, si la entrada sigue activa, entonces activa la salida.

La diferencia principal es la siguiente:

- TOF: activa de inmediato y retrasa el apagado.
- TON: espera primero y después activa.

En otras palabras, el TOF sirve para mantener una salida encendida durante un tiempo después de quitar la señal, mientras que el TON sirve para encender una salida solamente si la señal se mantiene activa durante cierto tiempo.

7. CONTACTO %I0001 – BOTÓN DE ARRANQUE

El primer elemento de la línea es un contacto normalmente abierto asociado a la entrada física %I0001.

La entrada I1 corresponde a un botón físico normalmente abierto. Esto significa que, cuando el botón no está presionado, el contacto está abierto y no permite el paso de corriente lógica.

Cuando el operador presiona el botón I1, el contacto %I0001 se cierra. Al cerrarse, permite que la corriente lógica llegue al bloque temporizador TON.

En este programa, I1 es la señal que inicia el conteo del temporizador. Sin embargo, no basta con presionar el botón rápidamente. El botón debe mantenerse presionado durante el tiempo configurado para que el temporizador pueda completar su conteo.

8. REGISTRO %R0001 DEL TEMPORIZADOR

El temporizador TON está asociado al registro %R0001.

En los PLC Horner, los registros tipo %R se utilizan para almacenar valores internos del programa. En el caso de un temporizador, el registro se utiliza para guardar información relacionada con el tiempo, el valor acumulado y el estado interno del bloque.

En este programa, %R0001 no representa una entrada física ni una salida física. Es una memoria interna que el PLC utiliza para administrar el temporizador TON.

El registro permite que el bloque temporizador sepa cuánto tiempo ha transcurrido desde que se activó la entrada y si ya se alcanzó el tiempo configurado.

9. BASE DE TIEMPO DEL TEMPORIZADOR

Debajo del nombre TON aparece la base de tiempo del temporizador. En este caso, la base de tiempo indicada es 0.1 segundos.

La base de tiempo define cuánto vale cada unidad escrita en el parámetro PT.

Algunos temporizadores pueden trabajar con diferentes bases de tiempo, por ejemplo:

- 0.01 segundos.
- 0.1 segundos.
- 1 segundo.

En este programa se utiliza una base de tiempo de 0.1 segundos. Esto quiere decir que cada unidad del valor PT equivale a una décima de segundo.

10. PARÁMETRO PT = 50

El parámetro PT significa Preset Time, es decir, tiempo preestablecido.

En este programa, PT tiene un valor de 50.

Como la base de tiempo es 0.1 segundos, el tiempo real del temporizador se calcula de la siguiente manera:

PT x base de tiempo

Sustituyendo los valores:

50 x 0.1 segundos = 5 segundos

Por lo tanto, el temporizador está configurado para esperar 5 segundos antes de activar su salida.

Esto significa que la entrada I1 debe permanecer activa durante 5 segundos continuos. Si el botón se suelta antes de que se cumpla ese tiempo, el temporizador vuelve a cero y la salida no se activa.

11. BOBINA INTERNA %M00001

A la derecha del temporizador se encuentra la bobina %M00001.

La dirección %M00001 corresponde a una marca interna del PLC. Las marcas internas son memorias lógicas que se utilizan para conectar partes del programa o para guardar estados internos.

En este programa, la marca M1 funciona como señal intermedia entre el temporizador TON y la salida física Q11.

Cuando el temporizador termina de contar los 5 segundos y su salida se activa, se energiza la bobina %M00001.

Al activarse M1, todos los contactos asociados a esa marca cambian de estado. En este caso, el contacto %M00001 de la segunda línea se cierra y permite activar la salida Q11.

12. SEGUNDA LÍNEA DEL PROGRAMA – ACTIVACIÓN DE Q11

La segunda línea del programa contiene un contacto normalmente abierto %M00001 y una bobina de salida física %Q0011.

El contacto %M00001 depende del estado de la marca interna M1.

Cuando M1 está apagada, el contacto %M00001 está abierto y no permite el paso de corriente lógica hacia Q11.

Cuando M1 se activa, el contacto %M00001 se cierra. Al cerrarse, la corriente lógica llega hasta la bobina %Q0011.

La salida %Q0011 corresponde físicamente a la lámpara amarilla del tablero. Por lo tanto, cuando Q11 se activa, la lámpara amarilla se enciende.

13. FUNCIONAMIENTO PASO A PASO DEL PROGRAMA

El funcionamiento completo del programa se puede explicar de la siguiente manera:

Paso 1: Estado inicial

Al inicio del sistema, el botón I1 no está presionado. El contacto %I0001 está abierto, el temporizador TON está reiniciado, la marca %M00001 está apagada y la salida %Q0011 también está apagada.

Por lo tanto, la lámpara amarilla conectada a Q11 permanece apagada.

Paso 2: Presionar el botón I1

Cuando el operador presiona el botón físico conectado a I1, el contacto %I0001 se cierra.

Al cerrarse este contacto, la corriente lógica llega al temporizador TON.

Paso 3: Inicio del conteo

Como el temporizador es tipo TON, la salida no se activa de inmediato.

En lugar de eso, el temporizador empieza a contar el tiempo configurado.

En este caso, el tiempo configurado es de 5 segundos.

Paso 4: Mantener presionado el botón

Para que el temporizador pueda completar su conteo, el botón I1 debe mantenerse presionado durante los 5 segundos.

Si el botón se mantiene presionado, el valor acumulado del temporizador va aumentando hasta llegar al valor PT.

Paso 5: Si se suelta el botón antes de tiempo

Si el operador suelta el botón antes de que pasen los 5 segundos, el contacto %I0001 se abre.

Cuando esto ocurre, el temporizador TON se reinicia. Su valor acumulado regresa a cero y la salida del temporizador no se activa.

En este caso, la marca M1 permanece apagada y la lámpara amarilla Q11 no enciende.

Paso 6: Cumplimiento de los 5 segundos

Si el botón I1 permanece presionado durante los 5 segundos completos, el temporizador llega al valor preestablecido.

Como PT vale 50 y la base de tiempo es 0.1 segundos, el tiempo total es de 5 segundos.

Al llegar a ese tiempo, el temporizador activa su salida.

Paso 7: Activación de la marca M1

Cuando el temporizador TON activa su salida, energiza la bobina interna %M00001.

Esto significa que la marca M1 se activa dentro del programa.

Paso 8: Cierre del contacto M1

Al activarse M1, el contacto normalmente abierto %M00001 de la segunda línea se cierra.

Esto permite que la corriente lógica avance hacia la bobina de salida %Q0011.

Paso 9: Encendido de la lámpara amarilla

Cuando la corriente lógica llega a %Q0011, se activa la salida física Q11.

Como Q11 corresponde a la lámpara amarilla, la lámpara amarilla se enciende.

Paso 10: Mantener la lámpara encendida

La lámpara amarilla permanece encendida mientras el botón I1 siga presionado y el temporizador TON siga manteniendo activa su salida.

En este programa no hay un enclavamiento que deje la lámpara encendida por sí sola después de soltar el botón. La condición depende de que la entrada I1 permanezca activa.

Paso 11: Soltar el botón después de encender la lámpara

Cuando el operador suelta el botón I1, el contacto %I0001 se abre.

Al abrirse el contacto, el temporizador TON se reinicia. Su salida se desactiva inmediatamente.

Paso 12: Apagado de la marca M1 y de Q11

Al desactivarse la salida del temporizador, la bobina %M00001 se apaga.

Cuando M1 se apaga, el contacto %M00001 de la segunda línea vuelve a abrirse.

Al abrirse ese contacto, la corriente lógica deja de llegar a %Q0011.

Como resultado, la salida Q11 se apaga y la lámpara amarilla se desactiva.

Paso 13: Sistema listo para iniciar nuevamente

Después de soltar el botón y apagarse la lámpara, el temporizador vuelve a cero y el sistema queda listo para repetir el ciclo.

Para volver a encender la lámpara, se debe presionar nuevamente I1 y mantenerlo presionado durante 5 segundos.

14. INTERPRETACIÓN DEL FLUJO DE CORRIENTE LÓGICO

En la primera línea del ladder, la corriente lógica inicia en el lado izquierdo. Si el botón I1 se presiona, el contacto %I0001 se cierra y permite que la señal llegue al temporizador TON.

El temporizador no deja pasar la salida inmediatamente. Primero espera a que se cumpla el tiempo configurado.

Cuando se cumplen los 5 segundos, el temporizador activa su salida y energiza la bobina %M00001.

En la segunda línea, el contacto %M00001 se cierra cuando M1 está activa. Al cerrarse, permite que la corriente lógica llegue a la bobina %Q0011.

La salida %Q0011 activa físicamente la lámpara amarilla.

Cuando I1 deja de estar activo, el temporizador se reinicia, M1 se apaga y Q11 también se apaga.

15. FUNCIÓN GENERAL DEL SISTEMA

La función general del sistema es activar la lámpara amarilla Q11 después de mantener presionado el botón I1 durante 5 segundos.

El temporizador TON evita que la lámpara se encienda con una pulsación rápida. Esto significa que el sistema necesita una señal continua durante el tiempo configurado.

Este comportamiento puede ser útil en automatización cuando se quiere confirmar que una condición se mantiene durante cierto tiempo antes de activar una salida.

Por ejemplo, si se quiere evitar que una señal momentánea active una lámpara, alarma o proceso, se puede usar un TON. Así, la salida solo se activa si la entrada se mantiene activa durante el tiempo suficiente.

16. COMPARACIÓN CON EL PROGRAMA TOF ANTERIOR

Este programa se parece al anterior porque ambos usan un temporizador, una marca interna M1 y la salida Q11 para controlar la lámpara amarilla.

Sin embargo, el comportamiento del temporizador es diferente.

En el programa con TOF:

- La salida se activaba inmediatamente al recibir la señal.
- El tiempo servía para retrasar el apagado.
- La lámpara se encendía al activar I1 y se apagaba después del retardo.

En este programa con TON:

- La salida no se activa inmediatamente.
- El tiempo sirve para retrasar el encendido.
- La lámpara se enciende solo si I1 permanece presionado durante 5 segundos.
- Al soltar el botón, la salida se apaga y el temporizador se reinicia.

La diferencia más importante es que el TOF retrasa el apagado, mientras que el TON retrasa el encendido.

17. APLICACIÓN EN AUTOMATIZACIÓN

El temporizador TON es uno de los bloques más utilizados en automatización industrial.

Algunas aplicaciones comunes son:

- Activar una lámpara después de que una condición se mantenga activa.
- Encender una alarma si una falla permanece durante varios segundos.
- Arrancar un motor después de un retardo de seguridad.
- Confirmar que un sensor detectó algo durante cierto tiempo.
- Evitar activaciones falsas por ruido eléctrico o pulsos muy cortos.
- Controlar la secuencia de arranque de una máquina.
- Dar tiempo a que un actuador llegue a una posición antes de activar otra salida.

En este programa, el TON se utiliza de forma didáctica para demostrar cómo se puede encender la lámpara amarilla Q11 después de mantener activo el botón I1 durante 5 segundos.

18. NOTA IMPORTANTE SOBRE EL TON

En un temporizador TON, la entrada debe permanecer activa durante todo el tiempo programado.

Si la entrada se desactiva antes de que se cumpla el tiempo, el temporizador se reinicia y no activa su salida.

Por eso, en este programa, el operador debe mantener presionado el botón I1. Una pulsación corta no es suficiente para encender la lámpara.

También es importante mencionar que, al soltar el botón, el temporizador se reinicia y la lámpara se apaga. Esto ocurre porque no existe un enclavamiento en el programa que mantenga activa la salida después de soltar el botón.

19. RESUMEN DEL PROGRAMA

Este programa ladder utiliza un temporizador TON asociado al registro %R0001.

El temporizador tiene una base de tiempo de 0.1 segundos y un valor PT de 50.

El tiempo total configurado se calcula así:

50 x 0.1 segundos = 5 segundos

El botón físico I1, representado por el contacto normalmente abierto %I0001, activa la entrada del temporizador.

Cuando I1 se presiona, el temporizador empieza a contar. Si el botón permanece presionado durante 5 segundos, el temporizador activa su salida y energiza la marca interna %M00001.

Al activarse M1, se cierra el contacto %M00001 en la segunda línea del ladder. Esto permite activar la salida física %Q0011.

La salida Q11 corresponde a la lámpara amarilla, por lo tanto, la lámpara amarilla se enciende después de que el botón I1 se mantiene presionado durante 5 segundos.

Cuando el botón I1 se suelta, el temporizador se reinicia, la marca M1 se apaga y la salida Q11 se desactiva. Como consecuencia, la lámpara amarilla se apaga.

En conclusión, este programa implementa un encendido con retardo usando un temporizador TON. La salida no se activa de inmediato, sino únicamente después de que la entrada I1 permanece activa durante el tiempo configurado.
