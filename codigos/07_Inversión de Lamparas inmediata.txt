EXPLICACIÓN DETALLADA DEL PROGRAMA LADDER CON SELECTOR, ENCLAVAMIENTO E INVERSIÓN DE COLOR
PLC HORNER HE-XC1E5 XL4 – CSCAPE 10.2

1. CONTEXTO GENERAL DEL PROGRAMA

El programa mostrado corresponde a un diagrama ladder, también llamado código en escalera. Este tipo de programación se utiliza en PLC, es decir, programadores lógicos programables, para automatizar procesos mediante entradas físicas, salidas físicas, marcas internas, contactos, bobinas, temporizadores, contadores y condiciones lógicas.

En este caso, el programa está desarrollado en Cscape 10.2, software utilizado para programar PLC Horner, específicamente de la serie HE-XC1E5 XL4.

El lenguaje ladder se interpreta como un circuito eléctrico de control. La corriente lógica fluye de izquierda a derecha. Primero se colocan las condiciones de seguridad y activación, después los contactos de control y enclavamiento, y al final de cada línea se colocan las bobinas que se van a activar.

Este programa permite realizar una inversión inmediata entre dos salidas usando un selector. Al activar la entrada I3, se enciende la lámpara amarilla Q11 y queda enclavada. Cuando se cambia el selector hacia la entrada I4, la lámpara amarilla se apaga inmediatamente y se enciende la lámpara roja Q12. De la misma forma, si se vuelve a activar I3, se apaga la roja y se vuelve a encender la amarilla.

La lógica se basa en enclavamientos y bloqueos cruzados entre marcas internas para evitar que las dos salidas queden activas al mismo tiempo.

2. ASIGNACIÓN GENERAL DE ENTRADAS Y SALIDAS DEL PLC

Para este PLC Horner HE-XC1E5 XL4 se consideran las siguientes entradas físicas del tablero:

- I1: Botón normalmente abierto, utilizado comúnmente como botón de arranque.
- I2: Botón normalmente cerrado, utilizado como botón de paro.
- I3: Entrada normalmente abierta asociada a una posición del selector.
- I4: Entrada normalmente abierta asociada a la otra posición del selector.
- I8: Botón de paro de emergencia normalmente cerrado.

También se consideran las siguientes salidas físicas para las lámparas del tablero:

- Q10: Lámpara verde.
- Q11: Lámpara amarilla.
- Q12: Lámpara roja.

En este programa se utilizan principalmente las salidas Q11 y Q12. Q11 corresponde a la lámpara amarilla y Q12 corresponde a la lámpara roja.

3. FUNCIÓN GENERAL DEL PROGRAMA

La función general del programa es controlar dos estados mediante un selector:

- Si se activa I3, se enciende la lámpara amarilla Q11.
- La lámpara amarilla queda enclavada.
- Si se cambia el selector hacia I4, se apaga inmediatamente Q11.
- Al activarse I4, se enciende la lámpara roja Q12.
- La lámpara roja también queda enclavada.
- Si se vuelve a activar I3, se apaga Q12 y se enciende Q11.
- El botón de paro I2 desactiva el sistema completo.

Este comportamiento se puede entender como una inversión de color inmediata. El sistema cambia de amarillo a rojo o de rojo a amarillo dependiendo de la posición del selector.

4. IMPORTANCIA DEL BOTÓN DE PARO I2

En la primera línea del programa aparece la entrada %I0002.

La entrada I2 corresponde al botón de paro. Físicamente, este botón es normalmente cerrado. Esto significa que, cuando el botón no está presionado, permite el paso de la señal. Cuando se presiona, abre el circuito y detiene el sistema.

En ladder, el botón de paro se coloca normalmente al inicio de la lógica. Esto se hace porque el paro debe tener prioridad sobre todo el sistema. Si el paro se activa, ninguna condición posterior debe mantener encendida una salida.

En este programa, I2 activa una marca interna de permiso, identificada como %M00010.

Mientras el paro I2 no esté presionado, la marca M10 se mantiene activa y permite que el resto del programa funcione.

Cuando se presiona el botón de paro I2, la marca M10 se desactiva. Al desactivarse M10, se corta el flujo lógico hacia las demás líneas del programa, apagando las salidas y desactivando los enclavamientos.

Por eso se dice que el paro siempre debe ir al inicio: porque funciona como condición general de seguridad o permiso para todo el circuito lógico.

5. MARCA %M00010 COMO PERMISO GENERAL DEL SISTEMA

La marca %M00010 funciona como una condición general de habilitación.

En la primera línea, el estado del botón de paro I2 se usa para activar o desactivar M10.

Después, en las líneas inferiores, M10 aparece como contacto al inicio de cada línea de control. Esto significa que, para que el sistema pueda activar la lámpara amarilla o la lámpara roja, primero debe estar activa la marca M10.

Si M10 está apagada, ninguna de las dos líneas puede funcionar. Por lo tanto:

- No puede activarse Q11.
- No puede activarse Q12.
- No pueden mantenerse los enclavamientos.
- El programa queda detenido.

Esta estructura es correcta porque coloca la condición de paro al inicio de la lógica general del sistema.

6. CONTACTOS, BOBINAS Y ENCLAVAMIENTO

En ladder es importante diferenciar entre contactos y bobinas.

Los contactos son condiciones. Sirven para permitir o bloquear el paso de corriente lógica. Un contacto puede estar asociado a una entrada física, una salida o una marca interna.

Las bobinas son elementos que se activan al final de una línea. Una bobina puede activar una salida física, una marca interna o una memoria del PLC.

En este programa se usan contactos para generar el enclavamiento. Esto significa que una vez activada una marca, un contacto asociado a esa misma marca se cierra y mantiene activa la línea aunque la entrada original ya no esté presente.

Por ejemplo, cuando I3 activa la marca M1, después el contacto M1 mantiene la línea activa. Eso es un enclavamiento.

También se usan contactos normalmente cerrados de marcas opuestas para evitar que las dos salidas estén encendidas al mismo tiempo. A esto se le puede llamar bloqueo cruzado o enclavamiento cruzado.

7. LÍNEA DE ACTIVACIÓN DE LA LÁMPARA AMARILLA Q11

La línea de la lámpara amarilla inicia con el contacto %M00010.

Esto significa que la lógica de la lámpara amarilla solo funciona si el sistema está habilitado y el paro I2 no ha sido presionado.

Después aparece la condición de activación asociada a %I0003.

La entrada I3 corresponde a una posición del selector. Cuando el selector se mueve hacia esa posición, I3 se activa y permite iniciar la lógica de la lámpara amarilla.

En paralelo con I3 aparece un contacto asociado a %M00001. Este contacto es el que realiza el enclavamiento.

El funcionamiento es el siguiente:

- Al activar I3, se permite el paso de corriente lógica.
- Se activa la marca M1.
- Al activarse M1, su contacto en paralelo se cierra.
- Cuando se suelta o cambia la condición inicial, el contacto M1 puede mantener activa la línea.
- Esto permite que la lámpara amarilla Q11 permanezca encendida.

Después de la condición I3/M1 aparece un contacto normalmente cerrado asociado a %M00004.

Este contacto sirve como bloqueo de la rama contraria. La marca M4 pertenece a la lógica de la lámpara roja. Cuando M4 se activa, abre este contacto y rompe el enclavamiento de la lámpara amarilla.

Al final de esta línea se activan las bobinas relacionadas con el estado amarillo:

- %M00001: marca de enclavamiento de la lámpara amarilla.
- %Q0011: salida física de la lámpara amarilla.
- %M00003: marca auxiliar que indica que el lado amarillo está activo y ayuda a bloquear la lógica contraria.

8. FUNCIÓN DE LA MARCA M1

La marca %M00001 se utiliza como memoria del estado amarillo.

Cuando I3 se activa, M1 se energiza. Al activarse M1, su contacto normalmente abierto en paralelo con I3 se cierra.

Esto permite que la lógica de la lámpara amarilla permanezca activa aunque la señal inicial del selector cambie momentáneamente.

Por eso se dice que M1 realiza el enclavamiento del estado amarillo.

Mientras M1 esté activa:

- La lámpara amarilla Q11 permanece encendida.
- La lógica del estado amarillo se mantiene.
- La marca auxiliar M3 puede indicar que el estado amarillo está activo.

9. FUNCIÓN DE LA MARCA M3

La marca %M00003 funciona como una marca auxiliar asociada al estado amarillo.

Su objetivo es participar en el bloqueo de la lógica contraria. En la línea de la lámpara roja aparece un contacto normalmente cerrado de M3.

Cuando el estado amarillo está activo, M3 se activa. Al activarse M3, el contacto normalmente cerrado de M3 en la línea roja se abre.

Esto ayuda a evitar que la lámpara roja Q12 se active al mismo tiempo que la lámpara amarilla Q11.

En otras palabras, M3 funciona como una señal de bloqueo para el lado rojo.

10. LÍNEA DE ACTIVACIÓN DE LA LÁMPARA ROJA Q12

La línea de la lámpara roja también inicia con el contacto %M00010.

Esto significa que la lógica de la lámpara roja solo funciona si el sistema general está habilitado por el botón de paro I2.

Después aparece la entrada %I0004.

La entrada I4 corresponde a la otra posición del selector. Cuando el selector se mueve hacia I4, se activa la lógica de la lámpara roja.

En paralelo con I4 aparece un contacto asociado a %M00002. Este contacto funciona como enclavamiento del estado rojo.

El funcionamiento es similar al de la línea amarilla:

- Al activar I4, se permite el paso de corriente lógica.
- Se activa la marca M2.
- Al activarse M2, su contacto en paralelo se cierra.
- El contacto M2 mantiene activa la línea aunque la señal inicial cambie.
- Esto permite que la lámpara roja Q12 permanezca encendida.

Después de la condición I4/M2 aparece un contacto normalmente cerrado asociado a %M00003.

Este contacto sirve para bloquear la lámpara roja cuando el estado amarillo está activo.

Al final de la línea se activan las bobinas correspondientes al estado rojo:

- %M00002: marca de enclavamiento de la lámpara roja.
- %Q0012: salida física de la lámpara roja.
- %M00004: marca auxiliar que indica que el lado rojo está activo y ayuda a bloquear la lógica amarilla.

11. FUNCIÓN DE LA MARCA M2

La marca %M00002 funciona como memoria del estado rojo.

Cuando se activa I4, M2 se energiza. Al activarse M2, su contacto en paralelo con I4 se cierra.

Esto genera el enclavamiento de la lámpara roja.

Mientras M2 esté activa:

- La salida Q12 permanece encendida.
- La lámpara roja se mantiene activa.
- La lógica del estado rojo queda memorizada.
- La marca auxiliar M4 puede bloquear el estado amarillo.

12. FUNCIÓN DE LA MARCA M4

La marca %M00004 funciona como una marca auxiliar asociada al estado rojo.

Su función principal es bloquear la lógica de la lámpara amarilla.

En la línea de la lámpara amarilla aparece un contacto normalmente cerrado de M4. Cuando la marca M4 se activa, ese contacto se abre.

Al abrirse el contacto de M4 en la línea amarilla, se rompe el flujo de corriente lógica hacia M1 y Q11.

Por lo tanto, cuando se selecciona el estado rojo, M4 ayuda a apagar inmediatamente la lámpara amarilla.

13. ENCLAVAMIENTO CRUZADO ENTRE AMARILLO Y ROJO

Este programa utiliza un enclavamiento cruzado.

El enclavamiento cruzado significa que cada estado tiene una condición para mantenerse activo, pero también tiene una condición que lo bloquea cuando se activa el estado contrario.

En este programa:

- M1 mantiene activa la lámpara amarilla.
- M2 mantiene activa la lámpara roja.
- M3 bloquea la lógica roja cuando está activo el estado amarillo.
- M4 bloquea la lógica amarilla cuando está activo el estado rojo.

La idea es que no puedan estar activadas ambas lámparas al mismo tiempo.

Cuando se activa I3, se favorece el estado amarillo. Cuando se activa I4, se favorece el estado rojo.

Este tipo de lógica es muy similar a la que se utiliza en inversión de giro, donde un motor no debe recibir al mismo tiempo la orden de giro hacia adelante y giro hacia atrás.

14. FUNCIONAMIENTO PASO A PASO DEL PROGRAMA

Paso 1: Estado inicial

El sistema inicia con el botón de paro I2 sin presionar.

Como I2 es un botón de paro normalmente cerrado, permite que la lógica de permiso active la marca M10.

Con M10 activa, las líneas de control quedan habilitadas.

Paso 2: Activación del selector en I3

Cuando el operador mueve el selector hacia la posición I3, la entrada %I0003 se activa.

La corriente lógica pasa por M10, después por I3, y llega a la lógica del estado amarillo.

Paso 3: Activación de M1 y Q11

Al cumplirse la condición de I3, se activa la marca %M00001.

También se activa la salida física %Q0011.

Como Q11 corresponde a la lámpara amarilla, la lámpara amarilla se enciende.

Paso 4: Enclavamiento del estado amarillo

Al activarse M1, el contacto M1 en paralelo con I3 se cierra.

Este contacto permite que el estado amarillo quede enclavado.

Gracias a este enclavamiento, Q11 puede permanecer encendida aunque la activación inicial de I3 ya no esté presente de forma momentánea.

Paso 5: Activación de la marca auxiliar M3

Al activarse el estado amarillo, también se activa la marca auxiliar %M00003.

Esta marca se utiliza para bloquear la lógica roja.

Cuando M3 está activa, el contacto normalmente cerrado de M3 en la línea roja se abre, impidiendo que Q12 se active al mismo tiempo.

Paso 6: Cambio del selector hacia I4

Cuando el operador mueve el selector hacia la posición I4, la entrada %I0004 se activa.

Al activarse I4, se inicia la lógica del estado rojo.

Paso 7: Activación de M4 para romper el estado amarillo

La lógica asociada a I4 activa la marca auxiliar M4.

Cuando M4 se activa, el contacto normalmente cerrado M4 ubicado en la línea amarilla se abre.

Al abrirse ese contacto, se rompe el enclavamiento de M1.

Paso 8: Apagado inmediato de Q11

Cuando se rompe el enclavamiento de M1, la salida Q11 se desactiva.

Como Q11 corresponde a la lámpara amarilla, la lámpara amarilla se apaga inmediatamente.

Paso 9: Activación de M2 y Q12

Una vez que el estado amarillo queda desactivado, la línea roja puede activarse correctamente.

Se activa la marca %M00002 y la salida física %Q0012.

Como Q12 corresponde a la lámpara roja, la lámpara roja se enciende.

Paso 10: Enclavamiento del estado rojo

Al activarse M2, su contacto en paralelo con I4 se cierra.

Esto permite que el estado rojo quede enclavado.

La lámpara roja Q12 permanece encendida hasta que se vuelva a seleccionar el estado amarillo o se presione el botón de paro.

Paso 11: Activación de M4 como bloqueo del estado amarillo

Mientras el estado rojo está activo, M4 mantiene abierto el contacto normalmente cerrado que se encuentra en la línea amarilla.

Esto evita que Q11 vuelva a encenderse al mismo tiempo que Q12.

Paso 12: Regreso del selector hacia I3

Si el operador vuelve a mover el selector hacia I3, la lógica ocurre en sentido contrario.

Se activa la línea amarilla, se activa M3, se bloquea el estado rojo, se rompe el enclavamiento de M2, se apaga Q12 y se vuelve a encender Q11.

Paso 13: Presionar el botón de paro I2

Si se presiona el botón de paro I2, se desactiva la marca M10.

Como M10 está al inicio de las líneas de control, se corta el permiso general del sistema.

Al apagarse M10:

- Se apaga Q11.
- Se apaga Q12.
- Se eliminan los enclavamientos.
- El sistema queda detenido.

15. INTERPRETACIÓN DEL FLUJO DE CORRIENTE LÓGICO

La corriente lógica siempre debe analizarse de izquierda a derecha.

Primero se revisa el permiso general del sistema por medio de M10, que depende del botón de paro I2.

Después, dependiendo de la posición del selector, se activa una de las dos ramas principales:

- Rama amarilla con I3, M1, M3 y Q11.
- Rama roja con I4, M2, M4 y Q12.

Cada rama tiene su propio enclavamiento:

- I3 activa M1, y M1 mantiene el estado amarillo.
- I4 activa M2, y M2 mantiene el estado rojo.

Cada rama también tiene su bloqueo cruzado:

- M4 bloquea la línea amarilla.
- M3 bloquea la línea roja.

Esto permite que el cambio de estado sea inmediato y que solo una lámpara permanezca encendida a la vez.

16. POR QUÉ LAS BOBINAS VAN AL FINAL

En ladder, las bobinas normalmente se colocan al final de la línea porque representan el resultado de las condiciones anteriores.

Primero se colocan los contactos, que son las condiciones de entrada, seguridad, bloqueo o enclavamiento.

Después, al final, se colocan las bobinas que se activan si todas las condiciones de la línea se cumplen.

En este programa, las bobinas finales son:

- M10 como permiso general.
- M1 como memoria del estado amarillo.
- Q11 como salida física de la lámpara amarilla.
- M3 como bloqueo auxiliar del estado amarillo.
- M2 como memoria del estado rojo.
- Q12 como salida física de la lámpara roja.
- M4 como bloqueo auxiliar del estado rojo.

Los contactos son los que se usan para enclavar, no las bobinas directamente. La bobina activa una marca, y después el contacto asociado a esa marca se utiliza para mantener el flujo lógico.

17. APLICACIÓN COMO INVERSIÓN DE COLOR

En este ejercicio, el programa se utiliza para hacer una inversión de color inmediata.

Esto significa que el sistema cambia entre dos lámparas:

- Amarillo con Q11.
- Rojo con Q12.

Al seleccionar I3, se enciende amarillo. Al seleccionar I4, se apaga amarillo y se enciende rojo.

El cambio es inmediato porque una rama bloquea a la otra mediante marcas internas.

18. APLICACIÓN COMO INVERSIÓN DE GIRO

Aunque en este programa se usan lámparas para visualizar el funcionamiento, la misma lógica puede adaptarse a una inversión de giro de motor.

Por ejemplo:

- I3 podría sustituirse por un sensor o por una orden de giro hacia adelante.
- I4 podría sustituirse por otro sensor o por una orden de giro hacia atrás.
- Q11 podría sustituirse por la salida que activa el giro en un sentido.
- Q12 podría sustituirse por la salida que activa el giro en el sentido contrario.

De esta forma, el programa podría controlar una inversión de giro inmediata usando la misma estructura de enclavamiento cruzado.

Sin embargo, en una aplicación real con motores es importante considerar protecciones adicionales, como contactores con enclavamiento eléctrico y mecánico, tiempos muertos entre cambios de giro, protecciones térmicas y condiciones de seguridad. Esto evita que el motor reciba órdenes opuestas al mismo tiempo o que se invierta el giro de forma peligrosa.

Para fines didácticos, este programa demuestra el principio lógico de inversión usando lámparas.

19. RESUMEN DEL PROGRAMA

Este programa ladder permite controlar dos salidas mediante un selector.

El botón de paro I2, que físicamente es normalmente cerrado, se coloca al inicio del programa como condición general de seguridad. Mientras I2 no esté presionado, se activa la marca M10, la cual habilita el resto de la lógica.

Cuando se activa la entrada I3 del selector, se enciende la lámpara amarilla Q11. Esta salida queda enclavada mediante la marca M1. También se activa la marca M3, que ayuda a bloquear la lógica de la lámpara roja.

Cuando se cambia el selector hacia I4, se activa la lógica del estado rojo. La marca M4 rompe el enclavamiento del estado amarillo, apagando inmediatamente Q11. Después se activa la marca M2 y se enciende la salida Q12, que corresponde a la lámpara roja.

Cuando se vuelve a seleccionar I3, ocurre el proceso contrario: se bloquea el estado rojo, se apaga Q12 y se vuelve a encender Q11.

El programa utiliza contactos de marcas internas para realizar el enclavamiento y el bloqueo cruzado. Las bobinas se colocan al final de las líneas y representan las salidas o memorias que se activan cuando las condiciones se cumplen.

En conclusión, este programa implementa una inversión inmediata entre dos estados: lámpara amarilla y lámpara roja. La lógica puede entenderse como una inversión de color mediante selector, pero también puede servir como base para una inversión de giro usando sensores y salidas de motor, siempre agregando las protecciones necesarias en una aplicación real.
